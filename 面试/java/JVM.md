JVM（Java Virtual Machine）内存结构在 Java 8 及之后版本中主要分为以下几个区域，每个区域有其特定的作用和生命周期。   
以下是 基于 **Java 8+ 的标准内存模型** （注意：Java 8 移除了永久代，引入了元空间）：

✅ **JVM 内存主要分为两大类：**

**一、线程私有区域（Thread-Private）**

每个线程独享，随线程创建而创建，随线程销毁而释放。

**1. 程序计数器（Program Counter Register）**

**• 作用：** 记录当前线程正在执行的字节码指令地址（行号）。
>如果执行的是 native 方法，则值为 undefined。
  
**• 特点：**
>唯一一个不会发生 OutOfMemoryError 的区域。
>
>线程私有，生命周期与线程一致。

**2. 虚拟机栈（Java Virtual Machine Stack）**

**• 作用：存储方法调用的栈帧（Stack Frame），每个方法调用都会创建一个栈帧，包含：**
>局部变量表（Local Variables）
>
>操作数栈（Operand Stack）
>
>动态链接（Dynamic Linking）
>
>方法出口（Return Address）

**• 异常：**
>栈深度超限 → StackOverflowError
>
>栈无法扩展（如 -Xss 设置过小且系统内存不足）→ OutOfMemoryError
>
**• 线程私有**
>⚠️ 注意：常说的“堆内存”和“栈内存”中的“栈”，就是指这个虚拟机栈。

**3. 本地方法栈（Native Method Stack）**

**• 作用：为 JVM 调用 native 方法（如 C/C++ 编写的 JNI 函数）服务。**
>有些 JVM（如 HotSpot）将它和虚拟机栈合并实现。
>
>同样可能抛出 StackOverflowError 或 OutOfMemoryError。

**二、线程共享区域（Shared by All Threads）**

**4. 堆（Heap） —— 最大、最重要的内存区域**

**• 作用：** 存放**几乎所有对象实例和数组**（JVM 规范规定对象必须在堆上分配，但 JIT 优化可能逃逸分析后栈上分配）。

**• GC 主战场：** 垃圾回收器主要管理此区域。

**• 结构（分代）：**

**• 新生代（Young Generation）：**
>Eden 区
>
>Survivor From / To（S0, S1）

**• 老年代（Old Generation）**

**参数控制：**
>-Xms：初始堆大小
>
>-Xmx：最大堆大小

**• 异常：** java.lang.OutOfMemoryError: Java heap space

**5. 方法区（Method Area） —— 逻辑概念，Java 8+ 由元空间实现**

**• 作用：** 存储**类的元数据信息**，包括：
>类结构（字段、方法、接口）
>
>运行时常量池（Runtime Constant Pool）
>
>静态变量（static variables）
>
>即时编译器（JIT）优化后的代码缓存等

**Java 8 之前**：实现为 **永久代（PermGen）**，位于堆内。

**Java 8 及以后**：改为 **元空间（Metaspace）**，使用**本地内存（Native Memory）**，不再受 JVM 堆大小限制。
>参数：-XX:MetaspaceSize, -XX:MaxMetaspaceSize

**异常：**
>Java 7 及以前：java.lang.OutOfMemoryError: PermGen space
>
>Java 8+：java.lang.OutOfMemoryError: Metaspace

>**💡 注意：** 字符串常量池在 Java 7 之后也从永久代移到了堆中。

**🧩 补充说明：运行时常量池（Runtime Constant Pool）**
>是方法区的一部分。
>
>存放编译期生成的各种字面量和符号引用（如 "hello"、类名、方法名等）。
>
>具有动态性：运行期间也可将新常量放入池中（如 String.intern()）。

---

**📊 总结表格**

|内存区域|线程私有/共享|作用|是否会发生 OOM|
|--|--|--|--|
|程序计数器|私有|记录字节码行号|❌不会|
|虚拟机栈|私有|存储方法栈帧（局部变量、操作数栈等）|✅ 会（StackOverflow / OOM）|
|本地方法栈|私有|支持 native 方法调用|✅ 会|
|堆（Heap）|共享|存放对象实例、数组|✅ 会（最常见 OOM）|
|方法区（元空间）|共享|存储类元数据、常量池、静态变量|✅ 会（Metaspace OOM）|

**🔚 关键记忆点**

**对象在堆上，方法在栈上。**

**堆 = 对象；栈 = 方法调用；方法区 = 类信息。**

**Java 8 没有永久代，只有元空间（在本地内存）。**

**字符串常量池在堆里（Java 7+）。**

理解 JVM 内存结构，是排查内存泄漏、调优 GC、分析 OOM 问题的基础。

**虚拟机栈**

在 Java 虚拟机（JVM）中，**栈帧（Stack Frame）** 是用于支持虚拟机进行方法调用和执行的数据结构。它是线程私有栈（Java Virtual Machine Stack）中的基本元素，每个方法从调用到执行完成的过程，都对应着一个栈帧在虚拟机栈中的入栈和出栈过程。

**1. 局部变量表 (Local Variables)**

**作用：** 用于存放方法参数和方法内部定义的局部变量。

**细节：**
>**存储内容：** 包括基本数据类型（boolean, byte, char, short, int, float, long, double）和对象引用（reference）。对象本身存储在堆中，这里只存指向堆的指针。
>
>**容量计算：** 以“变量槽（Slot）”为单位。long 和 double 类型占用 2 个 Slot，其他类型占用 1 个 Slot。
>
>**索引规则：** 索引从 0 开始。如果是实例方法（非 static），索引 0 默认存放 this 对象引用，随后依次是方法参数和局部变量。
>
>**生命周期：** 变量的作用域结束并不意味着 Slot 立即被回收，但逻辑上不再可用。Slot 可以被复用（例如两个互斥的局部变量可能共用同一个 Slot 以节省空间）。

**2. 操作数栈 (Operand Stack)**

**作用：** 作为方法执行过程中的工作区，用于存放计算过程中的中间结果和参与运算的操作数。

**细节：**
>**工作原理：** 基于“后进先出”（LIFO）原则。大部分字节码指令（如 iadd, ldc, invokevirtual）都是从操作数栈中弹出数据，运算后再将结果压入栈顶。
>
>**深度变化：** 方法刚开始时栈为空。随着字节码指令的执行，栈的深度会动态变化。
>
>**类型匹配：** 除了 long 和 double 占用 2 个栈单元外，其他数据类型占用 1 个栈单元。JVM 严格规定操作数栈中的数据必须与指令要求的类型严格匹配，否则会在编译期或运行期报错。
>
>**方法调用：** 在调用其他方法时，参数会被压入当前方法的操作数栈，然后传递给目标方法的局部变量表。

**3. 动态链接 (Dynamic Linking)**

**作用：** 指向运行时常量池中该栈帧所属方法的引用，用于支持方法调用过程中的动态绑定。

**细节：**
>**符号引用转直接引用：** 在 Class 文件中，方法调用通常使用符号引用（如类名、方法名、描述符）。在运行时，通过动态链接，将这些符号引用转换为直接引用（内存地址），以便直接调用方法。
>
>**多态支持：** 这是实现 Java 多态（重写/Override） 的关键机制。当调用一个虚方法（非 private, static, final, constructor）时，JVM 需要通过这个链接在运行时确定实际调用的具体实现版本（晚绑定）。
>
>**简化调用：** 使得每个栈帧不需要重新去常量池查找方法信息，而是直接持有指向常量池项的指针，提高执行效率。

**4. 方法返回地址 (Return Address)**

**作用：** 记录方法执行完成后，程序计数器（PC）应该跳转回的位置，即调用该方法的下一条指令的地址。

**细节：**
>**正常退出：** 如果方法是通过执行 return 指令正常退出的，返回地址会被用来恢复上层调用者的执行流。
>**异常退出：** 如果方法是因为抛出未捕获的异常而退出的，则不会使用这个返回地址来确定恢复点，而是通过异常处理表寻找匹配的异常处理器。
>**本质：** 它本质上保存了调用者栈帧中的 PC 寄存器值。当当前栈帧出栈（方法结束）时，PC 寄存器会被更新为这个值，从而让 CPU 继续执行调用者的代码。

**补充说明：附加信息**

除了上述四个核心部分，栈帧中通常还包含一些**附加信息**（这些信息在规范中没有强制定义具体位置，由虚拟机实现决定）：

>**调试信息：用于支持源代码行号映射，方便调试器定位错误行。
>**参数信息：在某些实现中，为了支持反射或调试，可能会额外存储参数的元数据。

**总结图解**

当一个方法被调用时：

>1.JVM 创建一个新的栈帧。
>2.局部变量表 初始化参数和局部变量。
>3.动态链接 指向常量池中的方法符号。
>4.返回地址 记录调用者的下一行代码位置。
>5.方法执行过程中，操作数栈 频繁进出数据进行计算。
>6.方法结束，栈帧出栈，根据 返回地址 恢复执行流，释放内存。
