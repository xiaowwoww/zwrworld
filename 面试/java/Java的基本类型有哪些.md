这是一个非常基础，但面试官很喜欢用来**“钓鱼”**的问题。

普通开发者只会背：“8 种，int, long, double...”
但对于 35 岁的架构师，你要回答出**“内存模型”**和**“设计陷阱”**。

Java 的基本数据类型（Primitive Types）一共有 **8 种**，可以分为 **4 大类**。

---

### 1. 四大类八种（标准答案）

| 分类 | 类型 | 字节 (Byte) | 位 (Bit) | 默认值 | 关键注意点 (面试坑点) |
| --- | --- | --- | --- | --- | --- |
| **整数型** | **byte** | 1 | 8 | 0 | **-128 ~ 127**。文件流处理常用。注意溢出问题。 |
|  | **short** | 2 | 16 | 0 | 现在的 Java 项目很少用，主要是兼容老 C 库。 |
|  | **int** | 4 | 32 | 0 | **整数默认类型**。 |
|  | **long** | 8 | 64 | 0L | 必须加后缀 **`L`**（小写 `l` 容易看成 `1`，规范要求大写）。 |
| **浮点型** | **float** | 4 | 32 | 0.0f | 必须加后缀 **`f`**。**精度丢失严重**，千万别用来算钱！ |
|  | **double** | 8 | 64 | 0.0d | **浮点默认类型**。精度比 float 高，但依然会丢精度。 |
| **字符型** | **char** | 2 | 16 | '\u0000' | 存的是 Unicode 码点。**本质上是一个无符号整数** (0 ~ 65535)。 |
| **布尔型** | **boolean** | *未定义* | *未定义* | false | **JVM 规范未明确大小**。在栈中通常按 `int` (4字节) 处理，在数组中按 `byte` (1字节) 处理。 |

---

### 2. 架构师视角的 3 个“死亡陷阱”

面试官问完基本类型，紧接着就会问下面这三个问题，答不上来就减分：

#### 陷阱一：关于“钱”的计算

* **问：** “我们要算 1.0 - 0.9，用 double 还是 float？”
* **答：** **都不能用！**
* `1.0 - 0.9` 在 double 里算出来是 `0.0999999998...`。
* **解决方案：** 涉及到金额计算，**必须**使用 **`java.math.BigDecimal`**，且构造时要传 **String** 参数（`new BigDecimal("0.9")`）。



#### 陷阱二：自动装箱与拆箱 (Autoboxing/Unboxing)

* **问：** “`Integer a = 100; int b = 100;` 它们相等吗？会有什么性能问题？”
* **答：**
* **性能：** 基本类型存栈（Stack），包装类型（Integer）存堆（Heap）。频繁转换会产生大量临时对象，增加 GC 压力。
* **NPE 风险：** `Integer` 可以是 `null`，但 `int` 不行。如果 `Integer a = null`，你直接 `int b = a;`，JVM 会自动拆箱调用 `a.intValue()`，瞬间报 **NullPointerException**。**这是生产环境最常见的 Bug 之一。**



#### 陷阱三：Integer 缓存池 (Cache)

* **问：**
```java
Integer a = 127, b = 127;
Integer c = 128, d = 128;
System.out.println(a == b); // ?
System.out.println(c == d); // ?

```


* **答：** 结果是 **true** 和 **false**。
* **原理：** Java 为了优化，把 **-128 到 127** 之间的整数缓存到了 `IntegerCache` 里。
* `a` 和 `b` 都在缓存范围内，指向同一个对象，所以 `==` 为 true。
* `c` 和 `d` 超出了缓存，是两个 `new` 出来的不同对象，所以 `==` 为 false（必须用 `.equals()`）。



---

### 📝 总结

回答这个问题的满分姿势：

1. **先列举**：8 种类型，byte/short/int/long/float/double/char/boolean。
2. **补细节**：boolean 大小未定，long 要加 L。
3. **上高度**：**“但在实际架构设计中，我更关注 BigDecimal 的精度问题，以及 Wrapper 类拆箱时的 NPE 风险。”**

这一下子就把你和刚培训出来的应届生区分开了。😎
