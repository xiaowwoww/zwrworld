过去数据大了就得分库分表，导致代码极难写（比如跨表分页、分布式事务）。现在的趋势是：把复杂性交还给数据库底层，解放 Java 程序员。

前沿代表：TiDB、OceanBase (阿里)、GaussDB (华为)。

核心优势（HTAP）：

它在逻辑上就是一张完整的表，你写代码就像连单机 MySQL 一样，直接写 SELECT 和 JOIN。

但它的物理底层分布在几十台服务器上，容量无限扩展。

HTAP (混合事务/分析处理)：它不仅能扛住极高并发的写入（比如双十一、银行结算），还能直接在同一套数据上跑极其复杂的实时数据分析，不需要再把数据导给别的系统。

你的现状关联：因为国家的“信创”要求，建行等大行正在疯狂将 Oracle/MySQL 替换为国产的 OceanBase 或 GaussDB。你进去后，极有可能会直接使用这些新型数据库。
