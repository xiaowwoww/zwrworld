### 🔥 面试题一：MySQL 索引底层

**Q：为什么 MySQL InnoDB 引擎的索引要用 B+ 树，而不是 B 树、Hash 或者红黑树？**

**A（满分回答逻辑）：**
这也是一道考察“数据结构与磁盘 IO”结合能力的题。

1. **排除 Hash：**
* Hash 查询虽然是 ，但它**不支持范围查询**（如 `> 100`，`between A and B`）。业务场景中范围查询太常见了，所以 Hash 不合适。


2. **排除红黑树 (BST)：**
* 红黑树是二叉树，随着数据量增加，**树的高度（Height）会很高**。
* 数据库的数据存在磁盘里，访问一个节点就是一次 **磁盘 I/O**。树越高，I/O 次数越多，性能越差。


3. **排除 B 树 (B-Tree)：**
* B 树的**每个节点都存储数据（Key + Value）**。
* MySQL 的一页（Page）默认 16KB。如果节点存了数据，那么一页能存的 Key 就变少了。
* 这意味着树会变高，增加了磁盘 I/O 次数。
* 而且 B 树的叶子节点之间没有指针，**范围查询（Range Scan）需要回溯**，效率低。


4. **选 B+ 树（最终赢家）：**
* **矮胖：** 只有叶子节点（Leaf Node）存数据，非叶子节点只存索引（Key）。一页能存更多的 Key，树的高度通常只有 2-4 层，**3 次 I/O 就能查到千万级数据**。
* **扫库快：** 所有叶子节点通过**双向链表**连在一起。范围查询（`SELECT * FROM user WHERE id > 100`）只需要找到起点，顺着链表读就行，不用回溯。

### 🔥 面试题二：Redis 分布式锁（生产级）

**Q：如何用 Redis 实现分布式锁？如果业务执行时间太长，锁过期了怎么办？（Redisson 看门狗机制）**

**A（满分回答逻辑）：**
别只答 `SETNX`，那是初级水平。要答出 **Redisson** 的原理。

1. **基础实现：**
* 使用 `SET key value NX PX 30000` (原子命令：不存在则设置 + 过期时间)。
* 解锁时，必须校验 Value（UUID）是不是自己的，防止删了别人的锁（用 Lua 脚本保证原子性）。


2. **痛点（锁过期）：**
* 假设锁设置了 10秒，但业务跑了 15秒。
* 第 10秒 时锁自动释放了。
* 第 11秒 线程 B 进来了。**此时并发安全失效！**


3. **解决方案 —— 看门狗（Watch Dog）：**
* 我们使用 **Redisson** 框架。
* Redisson 获取锁时，如果不指定 leaseTime，默认给 30秒。
* 它会在后台启动一个**定时任务（TimeTask）**，每隔 10秒（leaseTime / 3）检查一下：“哎，线程 A 还在跑吗？”
* 如果还在跑，就**自动续期**，把过期时间重新设为 30秒。
* 这样就保证了：**只要业务不跑完，锁就永远不过期；如果机器宕机，看门狗也挂了，锁到期自动释放，不会死锁。**

### 🔥 面试题三：Spring 循环依赖

**Q：Spring 是如何解决 Bean 的循环依赖问题的？（三级缓存）**

**A（满分回答逻辑）：**
这个问题专门筛掉没看过源码的人。

1. **场景：** A 依赖 B，B 又依赖 A。
2. **核心机制：三级缓存（Three-Level Cache）。**
* **第一级（singletonObjects）：** 存**成品** Bean（完整的）。
* **第二级（earlySingletonObjects）：** 存**半成品** Bean（实例化了，但还没注入属性）。
* **第三级（singletonFactories）：** 存 **Bean 工厂（Lambda）**，用来生成半成品。


3. **流程破局：**
* **A 创建：** A 实例化（new A），把 A 的工厂放入**第三级缓存**。
* **A 注入 B：** 发现需要 B，去创建 B。
* **B 创建：** B 实例化，B 注入 A。
* **B 找 A：** B 去缓存找 A。一级没有，二级没有，去**第三级**拿到 A 的工厂，调用工厂得到 A 的**早期引用（半成品）**。
* **关键点：** B 拿到了 A 的引用（虽然 A 还没填好属性，但地址有了）。B 完成初始化，放入一级缓存。
* **A 继续：** B 好了，A 也就注入成功了，A 完成初始化，放入一级缓存。


4. **限制：**
* 只能解决 **Singleton（单例）** 的循环依赖。
* 只能解决 **Setter 注入** 的循环依赖（构造器注入无法解决，因为无法实例化）。
