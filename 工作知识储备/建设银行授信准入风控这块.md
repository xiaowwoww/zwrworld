一、业务本质：什么是“授信准入风控”？

简单说，就是在银行决定是否给客户借钱（发放贷款、提供信用卡额度等）以及借多少钱之前，进行的一系列自动化判断和决策过程。你的工作就是开发和维护实现这些自动决策的系统。
•   授信：指银行向客户提供信用的行为，包括贷款、信用卡、担保等。

•   准入：是门槛筛选，判断客户是否有资格进入后续的审批流程。比如，年龄、身份、基本信用记录是否满足最低要求。

•   风控：是风险评估，在准入基础上，进一步预测客户未来的违约概率，并决定授信的额度和利率。

二、你将接触的核心系统与技术组件

作为Java开发，你主要参与的是支撑上述业务的风控决策引擎系统及其周边模块。典型技术栈和组件包括：

1.  风控决策引擎：
    ◦   核心：通常是一个高性能、可配置的规则引擎。你可能使用开源方案（如Drools, Easy Rules）或行内自研引擎。

    ◦   你的工作：开发和维护规则模型。规则是“如果…那么…”的逻辑（例：如果(客户年龄<22 或 >60) 那么 拒绝准入）。你需要将这些业务规则转化为可执行的代码或配置。

2.  数据服务与集成：
    ◦   内部数据：通过微服务（Spring Cloud/Dubbo） 调用客户信息、账户交易、历史信贷等内部系统。

    ◦   外部数据：集成人行征信系统（信用报告）、公安身份验证、工商信息、第三方大数据公司（反欺诈、多头借贷数据）等。这里涉及大量的API调用、数据解析和格式化。

3.  模型服务集成：
    ◦   风控不仅靠规则，也靠机器学习模型（评分卡、反欺诈模型、申请评分模型等）。

    ◦   你的工作：作为Java开发，通常不直接训练模型，但需要开发服务接口，调用数据科学团队发布的模型服务，获取客户的信用评分、欺诈概率等，并将结果送入规则引擎进行综合决策。

4.  流程与工作流：
    ◦   整个授信流程可能由工作流引擎（如Activiti, Flowable） 驱动，串联起数据采集、规则执行、人工复核等环节。

三、你需要快速掌握的业务知识

1.  关键业务术语：
    ◦   C卡/A卡/B卡：申请评分卡、行为评分卡、催收评分卡。你将主要与A卡相关。

    ◦   反欺诈：识别伪造身份、团伙骗贷等行为。

    ◦   多头借贷：客户同时在多家机构借贷。

    ◦   规则集、规则链、决策流、决策表：规则引擎的核心概念。

    ◦   白名单/黑名单/灰名单：不同的客户名单，对应不同的风控策略。

2.  核心业务流程（简化）：

    客户申请 -> 数据采集（内外部）-> 规则引擎执行（准入+风险规则）-> 模型评分 -> 综合决策（自动拒绝/通过/转人工）-> 输出决策结果（额度、利率、期限）。

    ◦   主动与数据提供方（其他团队）确认接口契约和数据质量。

4.  代码与安全：
    ◦   所有决策逻辑必须可追溯、可解释。日志要详细记录“为什么做出了这个决策”（触发了哪条规则，用了哪个数据，得分多少）。

    ◦   性能至关重要：决策通常要求秒级甚至毫秒级响应。关注规则执行的效率，避免慢查询。

# 四、业务重点：授信准入风控到底有哪些关键字？

- 普惠金融（建行重点）

### 4. 审批流程
客户经理 → 初审 → 复审 → 审批 → 放款
你要做：**状态流转、权限控制、记录留痕**。
