进入建行这种架构庞大、权限森严的大型金融机构，第一天最忌讳的就是像无头苍蝇一样瞎摸索，或者因为配不好环境而产生急躁情绪。

作为经验丰富的老兵，第一天展现专业素养的最好方式，就是**目标明确地拿到核心资源，不问低级技术问题，只问环境和规范问题。**

这是一份为你量身定制的**“入职首日通关 Checklist”**，建议明天直接照着这个单子逐一打勾：

### 🎯 阶段一：打通物理与虚拟结界（上午核心任务）

这部分是重中之重，没有这些账号，你就是个“局外人”。

* **[ ] 1. 搞定云桌面与内网通信工具**
* 找 HR 或项目经理拿到云桌面（VDI）的登录账号和初始密码。
* 登录后，第一时间修改密码。
* 下载并登录行内指定的通讯软件（如企业微信内网版、蓝信或其他内部 IM 工具），并要求项目经理把你拉进**核心开发群**和**项目组大群**。


* **[ ] 2. 申请三大核心基础设施账号**
* **代码仓库（GitLab/SVN）：** 拿到账号，并在云桌面本地生成 SSH Key 配置上去。
* **数据库权限：** 申请开发环境（DEV）和测试环境（SIT）的只读/读写权限（通常需要走 OA 审批流，第一天务必把流程发起）。
* **项目管理与文档库：** 申请 Jira、Confluence（Wiki）或行内自研的项目管理平台账号。



### 🛠️ 阶段二：配置 Java 专属“生命线”（下午核心任务）

内网开发最大的痛点就是无法连接公网，这一步配置错了，一行代码都跑不起来。

* **[ ] 3. 拿到内网专属的 `settings.xml`（生死攸关！）**
* **老兵动作：** 绝不要自己去网上下。直接找带你的老员工（或同组的后端同事）要一份他们正在用的 Maven `settings.xml` 文件。
* **检查重点：** 确保里面的 `<mirrors>` 配置指向了建行内部的 Nexus 私服地址，而不是阿里云或 Maven 中央仓库。


* **[ ] 4. 索要核心环境配置文件**
* **老兵动作：** 找同事要一份开发环境的 `application-dev.yml`（或 Nacos/Apollo 的命名空间、账号密码）。
* **检查重点：** 拿到注册中心、配置中心、Redis、数据库的内部 IP 地址和访问凭证，确保本地 IDEA 能顺利连上这些中间件。


* **[ ] 5. 安装必备的离线插件**
* 因为连不上公网，如果你习惯用 Lombok、MyBatisX、SonarLint，向同事询问**内网公共网盘**的地址，下载这些插件的离线包（`.zip`）导入到 IDEA 中。



### 🤝 阶段三：人脉与业务对齐（贯穿全天）

技术是死的，人是活的。第一天要把“谁管什么”摸得清清楚楚。

* **[ ] 6. 认清“三大关键人物”**
* **直属技术主管（TL）：** 决定你绩效和派活的人。
* **带你的老员工（导师）：** 你未来几周问环境问题、业务逻辑的直接出口。请他喝杯咖啡或送个小零食破冰。
* **对应的产品经理/业务代表：** 搞清楚谁是提需求的人。


* **[ ] 7. 拿到两份核心文档**
* **API 接口文档地址：** YApi、Swagger 或内部平台的链接。
* **数据字典 / 数据库表结构文档：** 弄清楚核心表里那些 `0, 1, 2` 到底代表什么业务状态。

### 💡 精神与精力管理锦囊

* **接受“慢”的现实：** 银行的 OA 审批极慢，第一天你大概率会卡在某个权限审批节点上，面对着屏幕发呆。**不要烦躁，这就是外包的日常。** 趁着等审批的空档，静下心来翻翻刚拿到的文档，或者闭目养神，把这当作一种定力修行。
* **准点下班，去出出汗：** 第一天通常不会有实质性的开发任务，但大量的信息轰炸和陌生的压抑环境会极大地消耗你的精力。下班时间一到，收拾东西走人。晚上换上运动鞋，去熟悉的路线上跑个 3、4 公里。大汗淋漓之后，把你身体里的疲惫和第一天的新手紧张感彻底排空，晚上睡个好觉。

**一切准备就绪！一旦环境配好，第二天 TL 大概率就会让你拉下几十个微服务的代码仓库。面对这种几百万行、深不见底的金融系统源码，你想不想掌握一套“老兵专属的看代码心法”，让你在三天内就能顺藤摸瓜，看懂它最核心的资金流转脉络？**
